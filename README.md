# ssh-rabbit

> rabbit4种模式【工作队列(work-queue)、发布/订阅(扇出、fanout)、路由(direct)、主题(topic)】

> 说明

``` bash
# 1 work模式中
## 1.1 一个生产者对应多个消费者,但是只能有一个消费者获得消息!!!  同一个消息只能被一个消费者获取
## 1.2 生产者和消费者都需要申明队列，都不需要申明交换机

# 2 fanout模式中
## 2.1 一个消费者将消息首先发送到交换器,交换器绑定多个队列,然后被监听该队列的消费者所接收并消费.
## 2.2 生产者需要申明交换机但不要申明队列
## 2.3 消费者需要申明队列并绑定交换机（交换机不存在时，绑定会报错）

# 3 direct模式中
## 3.1 生产者将消息发送到direct交换器,在绑定队列和交换器的时候有一个路由key,
##     生产者发送的消息会指定一个路由key,那么消息只会发送到相应key相同的队列,接着监听该队列的消费者消费信息.
## 3.2 生产者需要申明交换机但不要申明队列
## 3.3 消费者需要申明队列并绑定交换机（交换机不存在时，绑定会报错）
## 3.4 direct与fanout不同的是：绑定队列到交换机时会指定routingKey

# 4 topic模式
## 4.1 上面的路由模式是根据路由key进行完整的匹配(完全相等才发送消息),这里的通配符模式通俗的来讲就是模糊匹配.
## 4.2 符号"#"表示匹配一个或多个词,符号"*"表示匹配一个词.
## 4.3 生产者需要申明交换机但不要申明队列
## 4.4 消费者需要申明队列并绑定交换机（交换机不存在时，绑定会报错）
## 4.5 topic与direct不同的是：topic是模糊匹配
```
